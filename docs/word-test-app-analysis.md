## 1. 网站定位

这是一个纯前端的单页单词测试应用，主要用于老师或学生导入重点单词和英文文章，然后生成选择题练习。网站没有后端，所有数据保存在当前浏览器的 `localStorage` 中。

当前产品核心目标：
- 管理一组英文单词和中文释义。
- 导入一篇英文文章，用文章内容辅助生成情景题和阅读题。
- 生成大屏选择题，适合电脑和平板练习。
- 自动记录答错题目。
- 将答错单词加入抗遗忘复习池。
- 统计累计答题数量、答对数量和正确率。

当前页面标题：`单词课测试软件`

当前宣传语：`先导入单词和文章，再生成大屏选择题与抗遗忘训练。`

## 2. 目标用户与使用场景

### 2.1 目标用户

主要用户可能是：
- 英语老师：导入课程重点词和课文，课堂上让学生做选择题。
- 学生：自己导入单词表，进行自测和错题复习。
- 家长或助教：为学生准备词汇练习材料。

### 2.2 主要使用场景

场景 1：课前准备
1. 老师打开网站。
2. 导入本节课的重点单词。
3. 导入相关英文文章。
4. 设置题目数量和题型。
5. 生成测试。

场景 2：课堂练习
1. 学生在大屏或平板上看题。
2. 选择答案。
3. 提交答案。
4. 系统即时显示正确/错误、正确答案和解释。
5. 继续下一题。

场景 3：课后复习
1. 学生进入错题本查看答错题。
2. 进入抗遗忘训练复习答错单词。
3. 查看统计页了解累计表现。

该页面用于批量录入单词。

### 页面内容

左侧主面板：

- 提示：`每行一个：英文, 中文。也支持用制表符或中文逗号分隔。`

### 输入格式

每行一个单词，支持以下分隔符：
- 英文逗号：`,`
- 中文逗号：`，`
- 制表符：Tab

示例：

```text
example, 例子
journey, 旅行
improve, 提高
```

解析规则：
- 每行会被 `trim`。
- 空行会被忽略。
- 如果一行拆分后少于 2 个字段，则该行无效。
- 第一个字段作为英文。
- 后面所有字段会用中文逗号拼回中文释义。

| 返回首页 | 次按钮 | 不保存当前输入，直接回首页 | 切换到 `home` |

### 异常提示

| 清空复习池 | 次按钮 | 清空所有复习记录 | 弹出确认框，确认后清空 |

### 无待复习词提示

如果没有到期复习词，点击“开始抗遗忘训练”会提示：

```text

```

### 清空复习池确认文案

```text

```

### 抗遗忘生成题目规则

系统取所有满足以下条件的复习词：
- `stage < 3`
- `nextAt <= 当前时间`

然后最多生成 16 道复习题。

每道题固定为：
- 题型：`抗遗忘训练 第 X 轮`
- 题干：选择英文单词的中文意思
- 选项：1 个正确中文 + 3 个错误中文

### 抗遗忘阶段规则

复习词数据包含：
- `stage`：复习阶段，初始为 0。
- `wrong`：错误次数。
- `nextAt`：下次复习时间。
- `createdAt`：创建时间。

答对时：
- `stage` 加 1，最高到 3。
- 根据新阶段设置下次复习时间。

当前延迟配置：

| 新 stage | 下次复习间隔 |
|---:|---|
| 1 | 5 分钟 |
| 2 | 30 分钟 |
| 3 | 24 小时 |

代码中还定义了 stage 0 对应 1 分钟，但答对后会先加到 stage 1，所以这个 1 分钟延迟在当前答对逻辑中基本不会被使用。

答错时：
- `stage` 重置为 0。
- `nextAt` 设置为当前时间，立即可复习。
- `wrong` 加 1。

## 6. 题型生成规则

当前系统支持 4 类题型，全部都是选择题。

### 6.1 看英文选中文

来源：词表。

题目结构：
- 题型：`看英文选中文`
- 大词：英文单词
- 题干：`请选择 “xxx” 的中文意思。`
- 正确答案：该单词中文释义
- 错误选项：其他词的中文释义 + 兜底中文干扰项

### 6.2 看中文选英文

来源：词表。

题目结构：
- 题型：`看中文选英文`
- 大词：英文单词
- 题干：`“xxx” 对应哪个英文单词？`
- 正确答案：英文单词
- 错误选项：其他英文单词 + 兜底英文干扰项

### 6.3 情景句选词

来源：文章句子 + 词表。

生成逻辑：
1. 将文章按英文句号、问号、感叹号分句。
2. 只保留长度大于 20 的句子。
3. 找到句子中第一个命中的导入单词。
4. 将该单词替换成 `________`。
5. 让用户根据句意选择正确英文单词。

题目结构：
- 题型：`情景句选词`
- 大词：命中的英文单词
- 题干：带空格的句子
- 补充说明：`根据句意选择正确单词。中文提示：xxx`
- 正确答案：英文单词
- 错误选项：其他英文单词 + 兜底英文干扰项

### 6.4 阅读选择题

来源：文章 + 词表。

当前阅读题比较简单，主要有两种：

第一种：文章主旨类固定题
- 题干：`根据文章内容，下面哪一项最符合文章？`
- 正确答案固定为：`文章围绕人物经历或主题展开，并包含导入的重点词。`
- 干扰项固定为做菜、无关联单词、天气预报数据。

第二种：文章中重点词释义题
- 如果文章中出现了某个导入单词，则生成该词的释义题。
- 题干：`文章中出现的重点词 “xxx” 最接近哪个中文意思？`

### 6.5 题目数量规则

系统会先构建所有符合条件的题目，再随机打乱，然后截取设置数量。

例如设置 18 题：
- 如果构建出超过 18 题，只取前 18 题。
- 如果构建出的题少于 18 题，实际题目数会少于 18。

## 8. 状态和业务流程

### 8.1 初次使用流程

```text
首页
  -> 导入单词
  -> 保存单词
  -> 导入文章
  -> 保存文章
  -> 设置并生成测试
  -> 生成测试
  -> 测试页答题
  -> 结果总结
```

### 8.2 错题流程

```text
提交答案
  -> 判断错误
  -> 写入 mistakes
  -> 普通测试模式下写入 memory
  -> 保存 localStorage
  -> 错题本展示
```

### 8.3 抗遗忘流程

```text
普通测试答错
  -> 单词进入复习池 stage=0
  -> 抗遗忘页显示待复习
  -> 开始抗遗忘训练
  -> 答对则 stage + 1 并推迟 nextAt
  -> 答错则 stage 重置为 0 并立即可复习
  -> stage >= 3 算作已巩固
```

### 8.4 统计流程

```text
每次提交答案
  -> answered + 1
  -> 如果正确 correct + 1
  -> 统计页显示累计正确率
```

注意：统计是累计统计，不是按测试批次统计。

## 10. 当前优点

- 单文件即可运行，部署和分享简单。
- 核心学习闭环完整：导入、测试、答题反馈、错题、复习、统计。
- 选择题大屏设计清晰，适合课堂场景。
- 错题自动记录，不需要学生手动整理。
- 答错单词自动进入抗遗忘池，形成复习闭环。
- 词表和文章都保存在本地浏览器，刷新后不会丢失。
- 基础响应式布局已经存在，小屏幕下会变成单列。

## 11. 当前问题与限制

### 11.1 产品功能限制

1. 没有用户系统
   - 不同学生无法区分数据。
   - 老师无法查看多个学生表现。

2. 数据只保存在当前浏览器
   - 换设备会丢失。
   - 清理浏览器数据会丢失。
   - 无法导出备份。

3. 没有批次概念
   - 统计只有累计值。
   - 不知道每一次测试的题目、用时、正确率。

4. 阅读题生成较弱
   - 当前主旨题是固定模板，不是真正理解文章。
   - 阅读题数量很少。

5. 没有拼写题、听力题、填空题
   - 当前只支持选择题。
   - 对单词掌握程度的检测较浅。

6. 没有题目难度设置
   - 不能按错误率、词频、掌握程度出题。

7. 没有手动编辑/删除单词
   - 单词只能整体重新保存。
   - 不能单独修改某个词。

8. 错题本不能重新练习单题
   - 只能查看错题，不能从错题本直接生成复习。

9. 抗遗忘规则过于简单
   - 间隔时间写死。
   - 没有显示每个词的下次复习时间。
   - 没有解释 stage 含义。

### 11.2 交互体验问题

1. 隐藏页面没有导航高亮
   - 导入单词、导入文章、设置页没有顶部导航激活状态。

2. 导入示例数据没有成功提示
   - 用户点击后只看到词表变化，不一定知道文章也已导入。

3. 保存文章允许空内容
   - 点击保存空文章也会回首页。

4. 测试页可以不答完直接完成
   - 用户可以跳过题目到最后点完成，结果只统计已提交题目。

5. 上一题/下一题没有禁用态
   - 第一题的“上一题”看起来可点，但点击后无变化。

6. 清空资料不会清空错题和复习池
   - 这是代码刻意设计，但用户可能困惑，因为错题里可能还显示已清空词表中的词。

7. 错题本和统计没有导出功能
   - 老师无法保存学生记录。

8. 题目生成没有预览
   - 用户不知道生成后各题型比例。

9. 没有键盘快捷键
   - 大屏课堂或快速练习时，A/B/C/D 和 Enter 提交会更高效。

### 11.3 技术限制

1. 单文件结构会越来越难维护。
2. 使用 `Math.random() - 0.5` 洗牌，随机性质量一般。
3. 所有状态是全局变量，后续扩展复杂功能会变难。
4. 文章分句规则只适合简单英文句号、问号、感叹号。
5. 词匹配使用英文单词边界，对短语、变形词、大小写以外的变化支持不足。
6. `localStorage` 容量有限，不适合大量文章、班级数据或长期记录。

## 12. 改进建议

### P0：最值得先做的改进

1. 增加导入/保存成功提示
   - 保存单词后显示“已保存 X 个单词”。
   - 导入示例数据后显示“已导入 8 个单词和 1 篇文章”。

2. 增加单词管理能力
   - 支持单个单词编辑。
   - 支持单个单词删除。
   - 支持重复单词检测。
   - 支持导入错误行提示。

3. 改进测试完成规则
   - 未答完时点击“完成”提示还有几题未提交。
   - 允许用户选择“继续答题”或“仍然结束”。

4. 错题本增加“重新练习错题”
   - 一键从错题生成测试。
   - 支持只练某个错题。

5. 统计增加“本次测试记录”
   - 每次测试保存一条记录：时间、题数、答对、正确率。
   - 统计页显示最近 N 次测试。

### P1：提升学习效果

1. 增加题型
   - 拼写题：看中文拼英文。
   - 填空题：文章句子挖空输入单词。
   - 判断题：英文和中文是否匹配。

2. 增加按掌握程度出题
   - 优先出错过的词。
   - 优先出低正确率词。
   - 已巩固词降低出现频率。

3. 改进抗遗忘页
   - 显示每个复习词的 stage、错误次数、下次复习时间。
   - 支持手动移除某个词。
   - 支持一键复习全部错词。

4. 改进阅读题
   - 根据文章生成更多题型：
     - 细节理解
     - 词义猜测
     - 句子排序
     - 主旨大意
   - 当前固定模板可以保留为兜底题。

5. 增加键盘操作
   - A/B/C/D 选择答案。
   - Enter 提交。
   - 左右方向键切换上一题/下一题。

### P2：提升老师使用效率

1. 导入导出
   - 导出词表为 CSV。
   - 导入 CSV。
   - 导出错题和统计。
   - 导出完整备份 JSON。

2. 班级/学生维度
   - 本地可以先做“学生档案”。
   - 每个学生有独立统计和错题。

3. 测试模板
   - 保存常用设置。
   - 例如“课堂快测 12 题”、“课后复习 30 题”。

4. 打印能力
   - 打印词表。
   - 打印测试卷。
   - 打印错题本。

### P3：视觉与可用性

1. 移动端优化
   - 顶部导航在手机上更紧凑。
   - 大题卡减少高度，避免频繁滚动。

2. 状态更明确
   - 第一题禁用“上一题”。
   - 已提交按钮改成禁用态。
   - 当前页面面包屑，例如“首页 / 导入单词”。

3. 空状态更有行动指引
   - 测试页空状态增加“去生成测试”按钮。
   - 错题本空状态增加“开始测试”按钮。
   - 抗遗忘空状态说明怎样产生复习词。

4. 视觉一致性
   - 当前黑线 + 荧光黄风格鲜明，可以保留。
   - 危险操作按钮建议更清晰地和普通按钮拉开距离。

## 14. 验收标准建议

后续修改网站时，可以用这些标准检查是否做对。

### 导入单词

- 输入合法词表后保存成功。
- 首页词表数量正确。
- 词表预览与输入一致。
- 无效格式会提示。
- 重复单词有处理策略。

### 导入文章

- 保存文章后状态显示已导入。
- 统计页能看到文章预览。
- 空文章保存有明确提示或允许清空，两者需要产品上明确。

### 生成测试

- 没有单词时不能生成。
- 没有选择题型时不能生成。
- 生成题目数量不超过用户设置数量。
- 每题有 4 个选项。
- 正确答案必须在选项里。

### 答题

- 未选择答案不能提交。
- 提交后不能修改答案。
- 答对答错状态清晰。
- 统计正确更新。
- 答错后进入错题本。
- 普通测试答错后进入抗遗忘池。

### 错题本

- 错题显示题型、时间、题干、用户答案、正确答案、解释。
- 清空错题需要确认。
- 最多保留 100 条。

### 抗遗忘

- 有到期词时可以开始训练。
- 无到期词时给出提示。
- 答对提升 stage。
- 答错重置 stage。
- 清空复习池需要确认。

### 统计

- 累计答题、累计答对、累计正确率计算正确。
- 重置统计需要确认。
- 重置统计不应影响词表、文章、错题和复习池。

## 15. 一句话总结

当前网站已经具备“词表导入 -> 生成选择题 -> 即时反馈 -> 错题记录 -> 抗遗忘复习 -> 累计统计”的完整基础闭环。下一步最值得做的是增强导入反馈、错题再练、本次测试记录、抗遗忘词列表和数据导出，这些改动能明显提升老师和学生的实际使用价值。
